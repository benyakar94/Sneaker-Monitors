Getting Started

On this page, you will find steps to install and set-up the monitors from scratch.

If you'd rather follow a video tutorial, I have made some below. I recommend first testing out the scripts locally (on your PC) to ensure that you have the webhooks and other details set up. Then you should set them up on a server to run continuously.

  • Set up Locally
  • Set up on a Server

Installing Python and PIP

The monitors are built using Python 3, so you will need to have this installed. You can check whether you have it installed using the command in Terminal or Command Prompt:
python --version

Windows: Download and install Python from the Microsoft Store.
MacOS: Download and install Python from the Python Website.
Linux: Install Python using the command:
sudo apt-get install python3.8


Ensure you have PIP installed as well. It is a Python package installer and will be used to install all the dependencies. Usually PIP is installed with Python, but to check if you have it installed, use the command in Terminal or Command Prompt:
pip --version

If you haven't got it installed, you can use the commands:
curl https://bootstrap.pypa.io/get-pip.py -o get-pip.py

Then:
Windows: py get-pip.py
MacOS/Linux: python get-pip.py

Downloading the Files

There are two ways to do this. You can either download the ZIP folder or clone the repo. Cloning the repo makes it easier to recieve updates.

1. Download ZIP
Click on the green Code button and click on Download ZIP. Then unzip this folder.

2. Clone
Use git to clone the repo using the command:
git clone https://github.com/yasserqureshi1/Sneaker-Monitors.git

Setting Up

Now we can set up the monitors to our specifications.

1. Install the dependencies
Using Terminal or Command Prompt navigate to the downloaded/cloned folder. This can be performed using the cd command, that allows you to change directories as follows: 

C:\Users\Desktop> cd Sneaker-Monitors-master
C:\Users\Desktop\Sneaker-Monitors-master

Then install the dependencies using PIP:
pip install -r requirements.txt


2. Edit the .ENV File
Now, we can started customising the monitor. For this, we will only need to use the .ENV file.

Webhook: Paste your Discord Webhook URL under the WEBHOOK variable. It should look like this:
WEBHOOK = "https://discord.com/api/webhooks/..."

Location Code: The location code gets free proxies for your location. Note that this needs to be filled in with a valid location code regardless of if you use your own, due to the way it was programmed. However, if you do use your own, the free proxies won't be used. A list of the location codes can be found here.
LOCATION = "GB"

(Optional) Proxies: Paste your proxy under the PROXY variable. There are two structures:
a) PROXY = "<proxy>:<port>"
b) PROXY = "<proxy_username>:<proxy_password>@<proxy_domain>:<port>"

There are many other ways to customise the monitor and some monitors will require further details to work. More details on this can be found here.

3. Run the Python file
Now we run the particular file. In Terminal or Command Prompt, navigate to the directory housing the particular Python file (this was mentioned previously using the cd command). Then use the following command to run the file:
python [monitor].py

If you are using a Linux server, you can run multiple monitors using Linux Screen. This is a terminal multiplexer, so allows you to have multiple Linux sessions within a single window. You can install Linux Screen with the command:
sudo apt install screen

To start a new named screen session, use the command:
screen -S [name of session]

To close a session but leave any processes running in the background, press:
CTRL+a D

If you want to list all the screens, use the command:
screen -ls

To quit a session (i.e. delete it), use the command:
screen -XS [session number you want to quit] quit

To resume a session, use the command:
screen -r [session name or number you want to resume]

What next?

Once set up, the monitors instally start monitoring for changes. All you need to do is wait for the website to release or restock a product, and you should get notified.

If you set these up on your local PC and they seem to work, I suggest using a server. A server will run the code continuously so that changes are always picked up. There are some free trials that you can use, before commiting to a permanent solution. These include:

  • Google Cloud
  • Amazon Web Services (AWS)
  • Microsoft Azure
